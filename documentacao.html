1. Introdução e Conceito do Jogo

Delivery Dash — Brasília Edition é um jogo 2D com visão top-down (de cima para baixo) onde o jogador controla Alex, um entregador de moto que precisa fazer entregas pelas avenidas de Brasília.

História

Em Brasília, a capital do caos urbano, você é Alex, o entregador mais rápido da cidade. Uma tempestade colossal paralisa a metrópole: ruas alagadas, trânsito engarrafado e pedestres desesperados. Sua missão: pilotar sua moto pelas avenidas do Eixo Monumental, desviando de obstáculos e fazendo entregas antes que o tempo acabe!

Objetivo

Coletar pedidos (📦) e entregar nos destinos (🏠), acumulando pontos e ganhando tempo extra, enquanto desvia de carros, buracos, pedestres e outros perigos.

Como se perde

  • Vidas: o jogador tem 5 vidas (❤️). Cada colisão com obstáculo tira 1 vida. Zero vidas = Game Over.
  • Tempo: começa com 35 segundos. Se o tempo acabar = Game Over.

Importante: Não utilizamos nenhuma biblioteca externa, framework ou engine de jogos. Todo o código foi escrito do zero usando apenas as tecnologias nativas do navegador. Isso mostra o entendimento dos fundamentos.

Por que essas tecnologias?

  • HTML/CSS/JS — São as 3 tecnologias base da web. Qualquer navegador roda sem precisar instalar nada.
  • Canvas API — Permite desenhar formas, textos e gráficos diretamente no navegador, sem precisar de imagens externas.
  • Sem bibliotecas — Tudo feito na mão, demonstrando conhecimento das bases.

3. Estrutura de Pastas e Arquivos

delivery-dash/ ├── index.html ← Página principal (telas do jogo) ├── documentacao.html ← Este documento ├── css/ │ └── style.css ← Todos os estilos visuais ├── js/ │ ├── config.js ← Configurações e constantes │ ├── jogador.js ← Controle do personagem (Alex) │ ├── obstaculos.js ← Carros, buracos, pedestres, motos, poças │ ├── entregas.js ← Sistema de pedidos e destinos │ └── game.js ← Loop principal + todos os sistemas └── assets/ ← Pasta para recursos futuros

Por que separar em arquivos?

Cada arquivo tem uma responsabilidade única. Isso facilita:

  • Manutenção: se algo der errado nos obstáculos, sabemos que o problema está em obstaculos.js
  • Organização: cada membro do grupo pode trabalhar em um arquivo diferente
  • Entendimento: é mais fácil entender 5 arquivos pequenos do que 1 arquivo gigante

4. HTML — Estrutura da Página

O arquivo index.html contém a estrutura de 3 telas que se alternam:

Tela ID O que contém
Tela Inicial tela-inicio Título, história do jogo, instruções, botão "Começar"
Tela do Jogo tela-jogo HUD (vidas, tempo, pontos), Canvas, controles mobile
Tela Game Over tela-gameover Mensagem de derrota, pontuação final, botão "Tentar Novamente"

Como as telas alternam?

Usamos a classe CSS escondido que aplica display: none. Quando o jogador clica em "Começar", o JavaScript remove essa classe da tela do jogo e adiciona na tela inicial.

// Exemplo: trocar da tela inicial para o jogo document.getElementById('tela-inicio').classList.add('escondido'); document.getElementById('tela-jogo').classList.remove('escondido');

O elemento Canvas

O <canvas> é o elemento HTML onde todo o jogo é desenhado. Ele funciona como uma tela de pintura — o JavaScript desenha cada frame nele.

<canvas id="canvas-jogo"></canvas> // No JavaScript, pegamos o contexto 2D para desenhar: let canvas = document.getElementById('canvas-jogo'); let ctx = canvas.getContext('2d');

7. jogador.js — Controle do Personagem

Este arquivo cuida de tudo relacionado ao Alex (o entregador).

Funções principais

FunçãoO que faz

Como funciona o movimento?

Usamos dois event listeners: keydown (tecla pressionada) e keyup (tecla solta). Isso permite movimento contínuo enquanto segura a tecla.

// Quando pressiona a tecla, marca como true document.addEventListener('keydown', function(evento) { teclas[evento.key] = true; }); // Quando solta, marca como false document.addEventListener('keyup', function(evento) { teclas[evento.key] = false; }); // A cada frame, verifica quais teclas estão true e move if (teclas.ArrowLeft) jogador.x -= jogador.velocidade;

Detecção de colisão (AABB)

Usamos a técnica AABB (Axis-Aligned Bounding Box) — verificamos se dois retângulos se sobrepõem comparando suas bordas:

function verificarColisao(obj1, obj2) { return ( obj1.x < obj2.x + obj2.largura && obj1.x + obj1.largura > obj2.x && obj1.y < obj2.y + obj2.altura && obj1.y + obj1.altura > obj2.y ); }
Para explicar ao professor: "Comparamos as 4 bordas dos dois retângulos. Se nenhuma borda está separada, significa que estão se tocando — ou seja, houve colisão."

Controles invertidos (poça d'água)

Quando o jogador passa em uma poça, as variáveis controlesInvertidos e framesInvertidos são ativadas. Durante 2.5 segundos, a tecla da esquerda move pra direita e vice-versa, simulando derrapagem.

Tipos de obstáculos

TipoVisualComportamentoQuando aparece
CarroRetângulo com janelas e rodas (10 cores)Desce pela pista, mais rápido que buracosSempre
BuracoElipse escuraDesce devagar (estático)Sem chuva

Como os obstáculos funcionam?

  1. São criados acima da tela (posição Y negativa)
  2. Descem a cada frame (simulando a moto avançando)
  3. Quando saem por baixo da tela, voltam ao topo com posição X aleatória
  4. A quantidade e velocidade aumentam conforme o nível

Cores aleatórias

Carros e motos recebem cores aleatórias de um array quando são criados, dando variedade visual.

9. entregas.js — Sistema de Entregas

Ciclo de entrega

  1. Um PEDIDO (caixa laranja pulsante) aparece na pista
  2. O jogador coleta o pedido (colisão) → mensagem "Pedido coletado!"
  3. Um DESTINO (marcador verde com casinha) aparece
  4. O jogador entrega no destino (colisão) → ganha pontos + tempo
  5. Timer pausa por 3 segundos (respiro)
  6. Novo pedido aparece (ciclo recomeça)

10. game.js — Loop Principal e Sistemas

Este é o arquivo central que conecta tudo. Contém o loop principal e todos os sistemas especiais.

O Loop Principal (Game Loop)

O conceito mais importante de qualquer jogo. É uma função que roda ~60 vezes por segundo e executa sempre na mesma ordem:

function loopPrincipal() { 1. Limpa o canvas 2. Desenha o cenário (estrada, monumentos) 3. Move e desenha obstáculos 4. Move e desenha o jogador 5. Verifica colisões 6. Verifica entregas 7. Desenha efeitos (chuva, neblina) 8. Atualiza HUD 9. requestAnimationFrame(loopPrincipal) → repete }
Para explicar ao professor: "O requestAnimationFrame é uma função do navegador que chama nossa função a cada frame (~60fps). Isso cria a ilusão de movimento, igual um desenho animado — vários quadros por segundo."

Sistemas implementados no game.js

SistemaComo funciona
Vidas5 corações. Colisão = -1. Coração aparece na pista para recuperar. 0 = Game Over.
Timer35s inicial. +4s por entrega. Pausa 3s ao entregar. Relógio aparece quando faltam 10s.
EscudoItem azul na pista. 6 segundos de invencibilidade. Aura azul pulsante ao redor.
LadrãoNível 7+. Moto preta aparece atrás. 2s de alerta. Tenta roubar o pedido. Jogador pode desviar.
BRTFaixa cinza "BRT" aparece por 20s. Ônibus amarelos passam rápido. Colisão = perde vida.
CenáriosCores mudam por nível: dia → entardecer → noite → chuva → neblina → tempestade.
Capa de chuvaAlex troca a jaqueta verde por capa preta quando chove.
MonumentosMinistérios, Congresso, Catedral e Palácio descem pelas laterais.

Cenário de Brasília

As laterais da pista exibem monumentos famosos de Brasília, desenhados com formas geométricas simples no Canvas:

  • Ministérios: retângulo com pilotis (colunas) e janelas azuis espelhadas
  • Congresso Nacional: torres gêmeas + cúpula do Senado + cúpula invertida da Câmara
  • Catedral: costelas curvas com vitrais azuis e cruz no topo
  • Palácio da Alvorada: fachada de vidro com colunas em formato de asa

11. Mecânicas de Jogo

Progressão de dificuldade

  • A cada 3 entregas, sobe 1 nível
  • Cada nível adiciona +1 obstáculo (máximo 8)
  • Velocidade dos obstáculos aumenta +0.25 por nível (máximo 5)
  • Novos tipos de obstáculo aparecem (motos no nível 8+)
  • Ladrões aparecem a partir do nível 7
  • Cenário muda: dia → noite → chuva → tempestade

Balanceamento

O jogo foi balanceado para ser desafiante mas jogável:

  • Máximo de 8 obstáculos na tela (sempre tem passagem)
  • Velocidade máxima de 5 (nunca fica impossível)
  • Timer pausa 3s após entrega (respiro)
  • Relógio de bônus aparece quando faltam 10s
  • Escudo aparece periodicamente
  • Coração de recuperação aparece após perder vida
  • Invencibilidade de 1.5s após levar dano

12. Conceitos de Game Design

O jogo aplica conceitos de game design que criam uma experiência viciante:

Loop de Gameplay

Jogar → Perder → Tentar de novo. O jogador sempre quer "mais uma tentativa" porque sente que pode ir mais longe.

Mecânica de Recompensa

Cada entrega dá feedback positivo imediato (pontos + tempo + mensagem). Isso ativa a dopamina do jogador.

Escala de Dificuldade

A dificuldade sobe gradualmente. O jogador não percebe que ficou mais difícil — ele se adapta. Quando perde, sente que "quase conseguiu".

Feedback Visual

  • Flash vermelho ao levar dano
  • Flash azul ao passar na poça
  • Jogador pisca quando invencível
  • Aura azul quando com escudo
  • Pedidos e itens pulsam para chamar atenção
  • ⚠️ piscando quando ladrão se aproxima

Elementos de Brasília

O jogo usa referências reais da cidade: monumentos (Congresso, Catedral, Ministérios), faixa exclusiva do BRT, e as avenidas do Eixo Monumental. Isso dá identidade e contexto local ao jogo.

13. Hospedagem (GitHub Pages)

O jogo está hospedado gratuitamente no GitHub Pages, que transforma um repositório Git em um site público.

Como funciona

  1. O código está no GitHub em: github.com/henrique-uniceplac/delivery-dash
  2. O GitHub Pages serve os arquivos como um site estático
  3. Qualquer pessoa pode jogar acessando: henrique-uniceplac.github.io/delivery-dash
  4. A cada git push, o site atualiza automaticamente

Ferramentas usadas

  • Git: controle de versão (histórico de alterações)
  • GitHub: repositório remoto na nuvem
  • GitHub CLI (gh): ferramenta de linha de comando para interagir com o GitHub
Para explicar ao professor: "Usamos Git para versionar o código (cada alteração fica salva no histórico) e GitHub Pages para publicar o jogo online gratuitamente, sem precisar de servidor."
